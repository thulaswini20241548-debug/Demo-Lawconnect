# LawConnect Backend v2.0

Full Express.js backend for the LawConnect AI Legal Assistant platform.  
Built for the SDGP Group CS-165 project at the Informatics Institute of Technology.

---

## Quick Start

```bash
# 1. Install dependencies
npm install

# 2. Create your .env file
cp .env.example .env
# Then open .env and add your GROQ_API_KEY

# 3. Start the development server
npm run dev

# 4. Verify it's running
curl http://localhost:3000/health
```

## API Reference

### `GET /health`
Returns server status, uptime, Groq key check, and all endpoint paths.

---

### `POST /api/chat`
Main LexAI conversation endpoint.

**Request body:**
```json
{
  "message": "What are my rights if my landlord evicts me?",
  "history": [
    { "role": "user",      "content": "previous question" },
    { "role": "assistant", "content": "previous answer"   }
  ]
}
```

**Response:**
```json
{
  "reply":      "Under the Rent Act No. 7 of 1972...",
  "confidence": "high",
  "reason":     "well-covered by Rent Act 1972"
}
```

- `history` is optional (omit for the first message).
- History is capped at 20 entries server-side.
- `confidence` is one of: `"high"`, `"medium"`, `"low"`.

---

### `POST /api/analyse`
Analyses the text of an uploaded legal document.

**Request body:**
```json
{
  "documentText": "This Tenancy Agreement is made between...",
  "filename":     "lease_agreement.pdf"
}
```

**Response:**
```json
{
  "analysis":   "**Document Type:** Residential Lease Agreement\n\n**Plain-Language Summary:**\n...",
  "confidence": "high",
  "reason":     "clear residential lease under Rent Act",
  "filename":   "lease_agreement.pdf"
}
```

- `documentText` must be at least 50 characters and under 15,000 characters (~10 pages).
- PDF text extraction happens in the **browser** via PDF.js — only plain text reaches the backend.
- Documents are **never stored**.

---

### `POST /api/wizard`
Receives the focused prompt generated by the Legal Situation Wizard and returns a concise, actionable answer.

**Request body:**
```json
{
  "prompt": "I was unfairly dismissed after working more than 5 years. What gratuity and compensation am I entitled to under TEWA 1971?",
  "area":   "Employment Law"
}
```

**Response:**
```json
{
  "reply":      "Under the Gratuity Act No. 12 of 1983...",
  "confidence": "high",
  "reason":     "directly covered by Gratuity Act 1983"
}
```

- `area` is optional but improves response quality.
- Wizard answers are capped at 600 tokens for conciseness.

---

### `GET /api/practice-areas`
Returns the static list of all 6 practice areas (mirrors `data/practice-areas.js`).

**Response:**
```json
{
  "practiceAreas": [
    {
      "id":          "employment-law",
      "icon":        "💼",
      "name":        "Employment Law",
      "description": "...",
      "query":       "employment rights under Shop and Office Employees Act Sri Lanka",
      "laws":        ["Shop and Office Employees Act No. 19 of 1954", "..."]
    },
    ...
  ]
}
```

---

### `GET /api/practice-areas/:id`
Returns a single practice area by ID.

**Example:** `GET /api/practice-areas/employment-law`

---

### `POST /api/practice-areas/explain`
Returns a LexAI deep-dive overview of a legal topic.

**Request body:**
```json
{ "topic": "employment rights under Shop and Office Employees Act Sri Lanka" }
```

**Response:**
```json
{
  "reply":      "## Employment Law in Sri Lanka\n\n...",
  "confidence": "high",
  "reason":     "well-established area of Sri Lankan law",
  "topic":      "employment rights under Shop and Office Employees Act Sri Lanka"
}
```

## Environment Variables

| Variable       | Required | Default       | Description                                    |
|----------------|----------|---------------|------------------------------------------------|
| `GROQ_API_KEY` | ✅ Yes   | —             | Your Groq API key from console.groq.com        |
| `PORT`         | No       | `3000`        | Port the server listens on                     |
| `FRONTEND_URL` | No       | `*`           | CORS allowed origin (set your domain in prod)  |
| `NODE_ENV`     | No       | `development` | `development` or `production`                  |

## Deployment

### Railway (recommended)
1. Push this folder to a GitHub repo
2. Create a new Railway project → Deploy from GitHub
3. Add environment variables in Railway dashboard
4. Railway auto-detects `npm start` and deploys

### Render
1. Create a new Web Service → Connect your repo
2. Build command: `npm install`
3. Start command: `npm start`
4. Add environment variables in Render dashboard

### After deploying
Update `BACKEND_URL` in your frontend's `ai/backend-client.js`:
```js
const BACKEND_URL = 'https://your-app.up.railway.app';
```

---

## Connecting the Frontend

The frontend `ai/backend-client.js` calls all these endpoints automatically.  
The only value you need to change is `BACKEND_URL` at the top of that file.

```
Development:  BACKEND_URL = 'http://localhost:3000'
Production:   BACKEND_URL = 'https://your-deployed-api.com'
```

## AI Model

All endpoints use **Llama 3.3 70B Versatile** via Groq.  
- Temperature: `0.3` (low, for consistent legal responses)  
- Model string: `llama-3.3-70b-versatile`  
- Free tier at [console.groq.com](https://console.groq.com)
